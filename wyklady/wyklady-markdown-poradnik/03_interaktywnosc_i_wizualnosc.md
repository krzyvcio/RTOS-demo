# 3) Jak sprawić, aby wykłady w Markdown były ciekawsze i bardziej „interaktywne”

## Interaktywność bez narzędzi
### Pytania kontrolne w tekście
```md
**Pytanie:** Co się stanie, jeśli opóźnienie wzrośnie o 5 ms?
**Odpowiedź:** Margines fazy spadnie, układ może wejść w oscylacje.
```

### Mini-zadania
```md
### Zadanie (5 min)
Zbierz 10 s danych błędu i policz FFT. Zaznacz 3 największe piki.
```

### „Wybierz ścieżkę”
```md
Jeśli jesteś na poziomie A: przeczytaj sekcję 1 i 2.
Jeśli jesteś na poziomie B: dodaj sekcję 3 (pułapki).
Jeśli jesteś na poziomie C: zrób zadanie i rozbuduj model.
```

## Wizualność w Markdown
Bez grafiki nadal możesz:
- używać tabel do porównań,
- używać bloków kodu do schematów ASCII,
- stosować konsekwentne wyróżnienia.

Przykład schematu:
```text
sensor -> filter -> controller -> drive -> plant
```

## Checklisty
- Dodaj co najmniej 1 pytanie i 1 mini-zadanie na wykład.
- Schemat ASCII często daje więcej niż 200 słów opisu.
