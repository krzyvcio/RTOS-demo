# 2) Jak projektować strukturę wykładu w Markdown, aby utrzymać uwagę

## Szkielet, który działa prawie zawsze
```md
# Tytuł
## Po co to (motywacja)
## Model (minimalny)
## Narzędzia (algorytmy/metody)
## Pułapki (co psuje wdrożenie)
## Checklisty
```

## Nagłówki: reguły praktyczne
- `#` tylko raz (tytuł).
- `##` to rozdziały, które czytasz w 2–4 min.
- `###` to „kroki” w rozdziale.

## Listy i bloki
Używaj:
- list wypunktowanych do checklist i porównań,
- bloków kodu do równania lub konfiguracji,
- cytatów `>` do TL;DR i ostrzeżeń.

Przykład:
```md
> Uwaga: Jeśli nie logujesz jitteru, debugujesz „na ślepo”.
```

## Przykłady „mikro-ramki”
Dobre wstawki:
- „Najczęstszy błąd”
- „Co mierzyć”
- „Jak to sprawdzić”

## Checklisty
- Każdy wykład ma identyczną końcówkę `## Checklisty`.
- Utrzymuj rozdziały krótkie i przewidywalne.
- Dla trudnego tematu dawaj najpierw intuicję, potem formalizm.
