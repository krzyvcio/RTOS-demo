# Alternatywne wersje: 3 „dusze” wykładowcy w Markdown

## Wykładowca-entuzjasta
Cel: dynamika i energia, bez lania wody.

Szablon:
```md
## Dlaczego to jest ekscytujące
Tu dzieje się magia: mały jitter potrafi wywrócić stabilność!

## Szybki eksperyment
Zrób FFT, znajdź pik, dodaj notch i zobacz różnicę.
```

## Wykładowca-mentor
Cel: prowadzenie krok po kroku.

Szablon:
```md
## Krok 1: Objaw
Widzisz oscylacje.

## Krok 2: Minimalny model
Dodaj opóźnienie jako fazę w pętli.

## Krok 3: Co mierzyć
WCRT, jitter, dropouty.
```

## Wykładowca-storyteller
Cel: narracja i metafory, ale nadal klarownie.

Szablon:
```md
## Historia z produkcji
System działał tygodniami, aż nagle zaczął oscylować.
Powód: telemetria wbiła się w budżet RT i jitter skoczył.

## Morał
Deterministyczność to nie „szybko”, tylko „przewidywalnie”.
```
